# CLAUDE.md

## Project

`gcp-lite-rs` — Lightweight HTTP client for Google Cloud Platform APIs. Codegen-driven, OAuth2/ADC auth.

Rust edition 2024.

## Build & Test Commands

```bash
cargo check
cargo clippy -- -D warnings
cargo nextest run --lib
cargo fmt --check

# Codegen
cd codegen && uv run python -m codegen.cli apply          # Generate + apply all APIs
cd codegen && uv run python -m codegen.cli apply --api compute  # Single API
cd codegen && uv run python -m codegen.cli apply --dry-run      # Preview

# Integration tests (local dev only, not CI)
cargo test --test integration {api} -- --ignored --test-threads=1 --nocapture
```

## Architecture: Three-Layer Design

| Layer | Path | Generated? | Edit? |
|-------|------|-----------|-------|
| **Types** | `src/types/*.rs` | Yes | **NEVER** — codegen overwrites |
| **Ops** | `src/ops/*.rs` | Yes | **NEVER** — codegen overwrites |
| **Test Support** | `src/test_support/*.rs` | Yes | **NEVER** — codegen overwrites |
| **API** | `src/api/*.rs` | Scaffold once | **YES** — hand-written wrappers |

## Critical Rules

### GENERATED CODE IS IMMUTABLE — ENFORCED BY HOOK

**Files in `src/types/`, `src/ops/`, and `src/test_support/` are generated by codegen and MUST NEVER be edited by hand.** A PreToolUse hook blocks any attempt.

If generated code has a bug:
1. **STOP immediately**
2. **Flag the issue** to the user
3. **Propose a manifest-level solution** (edit `codegen/manifests/*.toml`)
4. **Wait for user approval**

### Other Rules

1. **To change types or ops**: edit the TOML manifest, then run codegen
2. **NEVER overwrite existing `src/api/*.rs` files** — only create new or extend
3. **Manifests** are the single source of truth for generated code
4. **No builders** — use `Default` derive + struct literal pattern
5. **Integration tests are for local dev only** — CI runs `cargo nextest run --lib`

### Authentication

GCP: `gcloud auth login` + `gcloud auth application-default login`
Set `GCLOUD_PROJECT_ID` env var.
